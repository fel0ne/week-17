# Финальный проект: Сервис уведомлений (Вариант notifications-s05)

## Обзор проекта

Данный проект представляет собой распределенную микросервисную систему, предназначенную для управления устройствами и автоматической отправки оповещений пользователям по различным каналам связи (`channel`: email, telegram и т.д.).

Проект разработан студентом группы 431 (ID: s05) в рамках финального задания курса.

---

## Архитектурный стек

| Компонент | Описание |
|---|---|
| **Gateway (Шлюз)** | Nginx (Порт `8080`) — точка входа для внешних клиентов, маршрутизирует префикс `/api` |
| **Основной микросервис** | `notifications-svc-s05` (Порт `8151`) — обрабатывает REST, GraphQL и содержит gRPC-контракт. Основная сущность: `Notification` со строковым полем `channel` |
| **Вспомогательный микросервис** | `devices-svc` (Порт `8266`) — имитирует бизнес-логику создания устройств и триггерит отправку уведомлений |
| **База данных** | Встроенная In-Memory (списки Python) для автономности и быстрой демонстрации |

Для локального запуска и проверки всей системы вам понадобятся:

- **Docker**

## Инструкция по быстрому запуску

Система спроектирована по принципу «инфраструктура как код» и поднимается одной-единственной командой из корня директории `week-17`:

```bash
docker compose up -d
```

Эта команда в фоновом режиме скачает базовые образы, смонтирует директории сервисов, настроит проксирование в Nginx и запустит все связанные службы.

### Проверка запущенных контейнеров

Убедиться, что все три компонента успешно стартовали, можно командой:

```bash
docker compose ps
```

## Демонстрация работы (Проверка API)

### 1. Получение истории уведомлений (REST API через Gateway)

Запрос проходит через Nginx Gateway (порт `8080`) и перенаправляется на внутренний порт `8151`:

```bash
curl http://localhost:8080/api/notifications
```

### 2. Симуляция сквозного процесса (Бизнес-сценарий)

Отправьте POST-запрос на создание нового устройства. Сервис устройств зарегистрирует его и асинхронно (в блоке изоляции ошибок) вызовет сервис уведомлений:

```bash
curl -X POST http://localhost:8080/api/devices \
     -H "Content-Type: application/json" \
     -d '{"name": "ASUS Gaming Laptop", "serial": "FA506NF-431", "owner_id": 505}'
```

После этого вы можете снова проверить список уведомлений через REST — там появится новая запись с каналом `email` по умолчанию.

### 3. Интерактивная документация GraphQL

Прямой доступ к панели GraphQL Playground для гибких выборок и выполнения мутаций доступен по адресу:

```
http://localhost:8151/graphql
```

Пример запроса:

```graphql
query {
  notifications {
    channel
  }
}
```
